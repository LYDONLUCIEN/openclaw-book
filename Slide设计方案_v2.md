# Slide 大纲（v3 最终版）

> 核心叙事：从"成本取舍"转为"三个好特性的持续追求"
> 主线：确定性、完备性、便利性——每一步进化都在提升至少一个特性

---

## 逐页大纲

### S00 封面
《OpenClaw 入门指南：从理解到实战》

### S01 问题引入
三个核心问题：
- 这些概念都是什么？（Agent、RAG、ReAct、MCP...）
- OpenClaw 是什么？
- 为什么用不起来？

### S02 三个好特性
确定性、完备性、便利性——所有技术方案都在追求这三个特性
- 确定性：输出可靠可预测（需要开发投入）
- 完备性：场景覆盖广（需要建设投入）
- 便利性：使用简单高效（需要技术投入）
- 三角图：三个顶点=三个特性，边上标注=投入方向

### S03 职业图鉴
四种技术路线的特性加点：
- 传统代码（执行者）：确定95 完备10 便利85
- 工作流（调度者）：确定90 完备20 便利75
- 机器学习（判断者）：确定65 完备35 便利80
- 大模型（通才）：确定20 完备90 便利60

### S04 OpenClaw vs 大模型
三个特性全面领先——怎么做到的？→ 四个版本、八件装备

### S05 应届生入职 + 三个核心困难
LLM = 清华应届生，三个困难：
- 记不住（影响确定性）
- 不懂规矩（影响完备性）
- 不会做事（影响确定性）

### S07 v1.0 人工教学（绿装）
Prompt + Function-call
- 确定性↑ 完备性↑ 便利性·
- 交互式 Prompt 示例 + Function-call JSON 示例

### S08 v2.0 流程标准化（蓝装）
RAG + Workflow + Knowledge Graph RAG
- 确定性↑ 完备性↑ 便利性·
- 交互式 RAG 四步演示（文档分块→向量化→相似度检索→注入上下文）
- KG-RAG 三步演示（实体抽取→关系构建→图谱检索）

### S09 v3.0 自主调度（紫装）
ReAct + MCP
- 便利性↑↑ 完备性↑ 确定性↓
- ReAct 八步交互演示（含自动播放）
- MCP 统一协议：USB Hub 图 + 高德地图真实代码示例

### S10 v4.0 龙虾！（橙装）
Skills + Harness Engineering
- 三个特性同时提升
- Skills 详细结构：真实 SKILL.md 格式（YAML frontmatter + Markdown）
- scripts/ 和 references/ 的使用方式
- Harness 对比 Prompt Engineering：全链路工程化（Tools/Memory/Triggers/Instruction/Output/Channel）
- Harness 自我进化能力

### S11 OpenClaw 架构
实际架构图，四层堆叠：
- Gateway 网关层（请求路由、会话管理）
- 认知层（Context Assembly + Memory）
- 执行层（ReAct Loop + Skills）
- 触达层（Node + Channel）
- 点击每层展开内部组件详解

### S12 四个使用层级
- Level 1 人设对话：确定性·完备性↑
- Level 2 任务助手：完备性↑便利性↑
- Level 3 定时调度：便利性↑↑确定性·
- Level 4 自主决策：三个特性↑但需监管
- 每个层级标注适用场景

### S13 局限性
四个核心弱点：
- 模型幻觉：概率模型不是事实数据库
- 自主决策风险：缺少常识和风险意识
- Token 消耗：一句"你好"消耗约13,500 tokens（交互式模拟演示）
- 安全边界：环境污染、权限过大、数据泄露

### S14 怎么做好龙虾？
三种投入方向 + 适合/不适合场景
- 适合：高频重复、可接受<100%确定性、有成熟工具、多变规则
- 不适合：100%确定性要求、低频一次性、规则明确、成本预算有限

### S15 进化路线（氪金）
四条进化路线，能力值对比：
- 执行者→Claude Code/Codex：完备性+35
- 调度者→DeerFlow/Dify：完备性+35 便利性+13
- 判断者→Evolver/EvoMap：完备性+25
- 通才→OpenClaw/Hermes：确定性+35 便利性+22

### S16 实战：浏览器脚本自动化
六步人机协作闭环：
人定义目标→AI理解需求→AI截屏分析→AI生成脚本→人审核结果→AI修正迭代
关键洞察：人负责"做什么"和"够不够好"，AI负责"怎么做"和"做多少次"

### S17 实战：团队协作模式
四角色协同：产品经理、OpenClaw Agent、审核者、调度器
协作闭环：需求→方案→评审→迭代→定时执行
含团队使用模式示意图（可替换为实际截图/GIF）

### S18 实战：这个课件本身
人机协作的真实产物——数十轮迭代：
第1轮：需求描述→初版大纲
第2轮：叙事重构（不可能三角→三个好特性）
第3轮：替换为真实 SKILL.md 和 MCP 代码
第N轮：逐页精调文案、动画、配色
分工：人（审美/叙事/取舍/决策）、AI（代码/动画/初稿/批量修改）
金句："你可以外包你的思考，但不能外包你的理解。"

### S19 未来展望
- 已有：聚智平台、磐匠 Agent
- 在建：云虾沙箱、安全管控、可信 Skill 仓库
- 未来：统一管理平台、更强底座、开放生态
金句："Agent 不会取代人，但会用 Agent 的人会取代不会用的人。"

### S20 总结
三个核心结论：
1. 确定性来自工程，不来自模型
2. 三个特性是方向，不是终点
3. 用好 Agent 的前提是理解 Agent
金句："最好的 Agent 不是最强的 Agent，而是你理解得最深的那个。"
