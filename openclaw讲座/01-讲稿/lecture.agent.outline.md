## 讲座主线

**核心叙事**：2024-2026年AI概念井喷，概念太乱。我们不逐个解释概念，而是从大模型的**三个根本问题**出发，每个问题自然引出对应的技术解决方案，最终用OpenClaw作为实战案例拆解。

**三条问题线索**：
1. 🔵 **记忆问题**：上下文有限 → 渐进式披露 + 上下文压缩 + Harness工程
2. 🟢 **知识问题**：不懂业务 → RAG + 知识图谱 + Skills
3. 🟠 **行动问题**：没有手脚 → MCP协议 + 工作流 + Tools

**逻辑链**：概念乱象 → 三个问题 → 对应解决方案 → ReAct循环 → OpenClaw实战拆解 → 横向对比 → 降温 → 场景穿透 → 现状 → 展望

## 详细内容大纲

### Part 1: 引子（Slide 02-04）

- 抛出问题："这些概念，到底是什么关系？"
- 动画：混乱标签 → 收敛为三个问题域

**Slide 03 — 产品生态全景**
- 四层架构：LLM底座 → Agent平台（Dify/Coze/n8n）→ Agent产品（OpenClaw/Claude Code/Manus）→ 工具协议（MCP/APIs）
- 每个产品的定位和关系

**Slide 04 — 大模型三大核心问题**
1. 🔵 记忆问题：上下文有限，注意力稀释，信息过载
2. 🟢 知识问题：不懂公司业务，不知道流程规范
3. 🟠 行动问题：只能生成文字，不能调用系统工具
- → "解决这三个问题 = 大模型进化为AI Agent"

### Part 2: 三个核心问题（Slide 05-07）

**Slide 05 — 记忆问题**：注意力稀释 → 渐进式披露 + 上下文压缩 + Harness工程
**Slide 06 — 知识问题**：RAG解决"知道什么" + 知识图谱解决"知道关联" + Skills解决"知道怎么做"
**Slide 07 — 行动问题**：MCP解决"能不能调用" + Tools提供具体能力 + Skills解决"调用效率"

### Part 3: ReAct（Slide 08）
- 有了记忆+知识+行动 → 模型可以自主迭代（ReAct循环）
- 完整交互演示：Think → Act(含tool调用) → Observe → 循环
