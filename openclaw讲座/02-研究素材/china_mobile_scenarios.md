## 一、中国移动数字化现状概览

### 1.1 战略定位：从"+AI"到"AI+"的转变

中国移动锚定"创世界一流信息服务科技创新公司"的目标，提出加速业务从"+AI"（AI作为辅助工具）向"AI+"（AI作为核心驱动）转变[^28^][^29^]。在这一战略中，智能体被视为大模型落地的"最后一公里"。

**三大核心痛点驱动转型：**

| 痛点 | 具体表现 | 解决方向 |
|------|----------|----------|
| 开发门槛高 | 传统AI开发需要深厚算法背景，难以普及到业务一线 | 低代码/无代码平台 |
| 系统协同难 | 企业内部数千个异构IT系统，数据孤岛问题突出 | 统一智能体底座 |
| 安全风险大 | 智能体自主执行可能引发数据泄露或系统误操作 | 平台化安全管控 |

### 1.2 数字化转型的关键数据

- **17000+数字员工**：已上岗运营，年减负70万人天，赋能16个领域[^49^]
- **350个生产场景**：仅甘肃移动就落地350个数字员工场景，累计减负突破10万人天[^40^][^41^]
- **百万亿级Token日调度**：算网大脑4.0实现日调度百万亿级Token[^88^][^90^]
- **300+AI模型**：移动模型服务平台MoMA整合超300款业界主流AI模型[^26^][^27^]
- **1.9亿用户**：AI产品覆盖规模[^36^]

### 1.3 核心技术底座架构

中国移动构建了"算力-模型-应用"全链路技术体系[^26^]：

```
┌─────────────────────────────────────────────────────────────┐

│  │ （Agent开发）  │ │ （RPA执行）   │ │ （场景落地）          │  │
│  └──────────────┘ └──────────────┘ └──────────────────────┘  │

│                     模型层（大模型）                           │

## 二、聚智智能体平台：AI Agent的"超级工厂"

### 2.1 平台定位与战略意义

聚智智能体平台是中国移动全网统一的智能体开发与运行底座，于2024年10月正式发布[^55^]。其核心目标是让不懂代码的一线业务人员也能用上AI，确保数万个智能体在企业内部安全、有序地运行[^28^][^29^]。

**平台解决三大核心问题：**
1. **开发门槛高** → 提供低代码/无代码开发套件
2. **系统协同难** → 统一的能力汇聚与组件复用
3. **安全风险大** → 全生命周期安全管控

### 2.2 核心功能模块

聚智平台采用模块化、分层解耦架构，以Agent OS（智能体操作系统）为核心[^28^][^29^][^55^]：

#### （1）单智能体组装
- 通过**一句话**快速创建单智能体
- 建设模板库、多模型管理、知识管理
- 长短期记忆管理、任务编排
- 组件/工具管理，支持智能体快速组装

#### （2）多智能体协同
- 跨平台、跨架构体系的智能体协作模式编排
- 支持不同架构（LangGraph/Dify/SOP）的单智能体组合
- 协作流程、流程测试及执行结果评估
- 面向复杂业务提升应用规划水平

#### （3）丰富的通用组件
- 通用组件仓库，持续沉淀汇聚
- 统一组件复用，赋能智能体快速开发
- 提供丰富、高效的组件服务

#### （4）异构的专用工具
- 统一接入协议和环境
- 高效工具调用能力
- 持续沉淀拓展专用工具

#### （5）统一的开放超市
- 智能体超市：沉淀共享智能体资产
- 运营管理：全生命周期管控

#### （6）专业的RPA集成
- 深度融合磐匠RPA能力
- 实现"大脑+手脚"的完整闭环

### 2.3 记忆层设计（解决大模型"失忆"问题）

聚智平台的知识库和记忆系统设计独特[^28^]：

**RAG（检索增强生成）——外挂"海马体"**
- 通过向量数据库持久化存储企业私有知识
- 有效解决大模型"幻觉"问题
- 知识库模块存储了大量中国移动相关政策文件
- 弥补开源模型"不了解"企业内部信息的短板

**长期记忆**
- 记录用户偏好、历史交互和任务状态
- 支持跨越数天的任务上下文保持
- 按用户、应用、项目等多维度分级存储
- 显著提升平台记忆能力与效率

### 2.4 技术架构特点

聚智平台构建了"工业级智能体生产线"[^29^]：

```
┌─────────────────────────────────────────────────────────────┐
│                        应用构建工坊                           │
│   （低代码/高代码开发套件 + 多场景应用模板）                    │
├─────────────────────────────────────────────────────────────┤
│                      Agent OS（智能体操作系统）                 │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────────┐   │
│  │ 运行时服务 │ │ 协议支撑  │ │ 记忆管理  │ │ 工具调用管理  │   │
│  └──────────┘ └──────────┘ └──────────┘ └──────────────┘   │
├─────────────────────────────────────────────────────────────┤
│                      全栈基础设施支撑                          │
│       （算力调度 + 模型服务 + 安全防护 + 运维监控）              │

## 三、磐匠数字员工：RPA到智能体的进化

### 3.1 平台定位

"磐匠"数字员工平台是中国移动全自研的融合AI、RPA、iPaaS等技术的超自动化软件机器人平台[^51^][^66^]。从2019年开始建设运营，历经RPA→IPA→智能体三代演进[^172^][^173^]。

**核心能力：**
- 自动操作软件，智能识别图像、语音、文字
- 发送5G消息，实现语音外呼及闪信霸屏
- 具备判断、决策能力
- 提供无代码设计能力、一站式执行载体、可视化运营管控

### 3.2 从RPA到AI Agent的三阶段演进

磐匠平台的发展清晰展示了数字员工的进化路径[^117^][^172^][^173^]：

| 阶段 | 时间 | 关键词 | 核心技术 | 能力范围 |
|------|------|--------|----------|----------|
| **RPA** | 2019-2022 | 自动化 | 流程自动化、规则引擎 | 高频重复流程型工作 |
| **IPA** | 2022-2024 | 智能化 | RPA+AI+NLP+OCR | 非结构化数据处理 |
| **AI Agent** | 2024至今 | 人机协同 | 大模型+RPA+IDP+GSUT | 自主决策+执行闭环 |

### 3.3 磐匠智能体的独特优势

中国移动信息技术中心副总经理陈国在2024年世界人工智能大会上介绍[^172^][^173^]：

> "磐匠"数字员工智能体融合大模型、RPA、IDP（智能文档处理）、GSUT（通用语义理解）等技术，可以模仿我们日常操作自动化调用各类应用软件，智能化处理各类文档，相当于为大模型装上"眼睛"和"手脚"。

**四大技术融合：**
- **大模型**：提供认知决策能力（"大脑"）
- **RPA**：提供UI自动化执行能力（"手脚"）
- **IDP**：智能文档处理能力（"眼睛"）
- **GSUT**：通用语义理解技术，视频模型自主识别系统界面

**创新亮点——"观摩式"无代码生成：**
- 用户操作一遍业务即可生成一个智能体
- 依托GSUT视频模型技术，能自主识别系统界面、自主学习软件操作
- 动态规划和编排工作任务
- 提供手机端、PC端多入口协同

### 3.4 规模化应用成果

磐匠平台已在全集团实现大规模应用[^49^][^40^][^41^]：

| 指标 | 数据 |
|------|------|
| 上岗数字员工总数 | 17000+ |
| 年减负效益 | 70万人天 |
| 覆盖领域 | 16大领域 |
| 甘肃移动数字员工 | 超千人，350个场景 |
| 江西移动数字员工 | 近1600个 |
| 覆盖行业 | 9大行业 |

### 3.5 16大通用领域应用

磐匠数字员工平台涵盖的16大通用领域[^51^]：

1. **财务**：报表核算、发票信息稽核
2. **人力**：绩效考核管理、智慧薪酬核算
3. **法务**：合同审核、企业资质审查
4. **运维**：设备资产管理、网络运维管理
5. **市场**：渠道运营分析、用户运营分析
6. **客服**：投诉工单处理、客户精准营销
7. **党建**：纪要生成、三会一课检查
8. **采购**：订单管理、物流跟踪管理
9. **综合办公**：文档分类处理、邮件处理转发

## 四、九天大模型：自主可控的AI底座

### 4.1 为什么要做九天大模型

中国移动投入巨资构建覆盖全国的智能算力网络，可以理解为"为AI时代修建的高速公路系统"[^32^]。**九天大模型正是为这条高速公路"制造车辆"的核心工厂**[^32^]。

**核心驱动力：**
1. **创造需求，消耗算力**——大模型训练和推理需要海量算力
2. **释放算力网络价值**——将算力基础设施从成本中心转变为利润中心
3. **防止被管道化**——在AI时代保持核心竞争力
4. **自主可控**——央企"大国重器"，全栈国产化[^39^]

### 4.2 技术特点

九天大模型具有多项显著特点[^50^][^54^]：

| 特性 | 说明 |
|------|------|
| **全栈自主可控** | 从数据构建、预训练、微调、推理全链路核心技术自主可控 |
| **国际技术领先** | 视觉理解、多语言理解、编码和数学等方面表现突出 |
| **灵活部署** | 支持云端、边缘端、终端"云边端"部署 |
| **行业能力增强** | 针对通信、电力、运输、能源等行业专业知识增强 |
| **安全可信** | 完善的数据指令评估体系，人类价值观对齐 |

### 4.3 技术参数与版本演进

- **训练规模**：超2304台服务器、1.8万张AI加速卡[^39^]
- **最新版本**：九天善智多模态基座大模型（2024年10月发布），九天安全可信大模型JT4.0（2026年5月）[^54^][^90^]
- **适配能力**：适配11个厂家17款国产AI芯片，支持异构芯片间平滑转化和续训[^54^]
- **模型矩阵**：30款自研行业大模型（2024年10月发布）[^54^]

### 4.4 算网大脑：智能调度中枢

算网大脑是中国移动智算体系的核心调度系统[^92^][^96^]：

**算网大脑3.0/4.0核心能力：**
- "1+N"多智能体协同架构：1个指挥智能体+N个专家智能体
- 深度融合九天大模型
- 实现百万亿级Token日调度
- 800GB高吞吐能力
- 1ms-5ms-20ms三级低时延圈
- 纳管80+EFlops算力资源
- 1500个边缘智算中心升级

### 4.5 移动模型服务平台MoMA

2026年5月发布的MoMA平台标志着AI模型"超级入口"正式落地[^26^][^27^]：

**平台特点：**
- 承载九天大模型+整合超300款业界主流AI模型
- 独创模型联邦与智能路由技术
- 多源异构模型统一接入与能力输出
- 单位Token成本压降30%以上

### 4.6 智能体框架MobileClaw

MobileClaw是中国移动推出的移动版智能体框架[^26^][^89^]：

- 一键接入各类主流IM工具

## 五、内部办公场景深度分析

### 5.1 场景一：大量材料/文档处理

#### 场景描述
中国移动各级员工每天需要处理海量文档，包括政策文件、技术规范、合同协议、审计报告、经营分析材料等。这些文档往往篇幅长、专业性强、数量多。

#### 当前痛点
- 阅读一份50页的政策文件需要2-3小时
- 不同系统导出的数据格式不统一
- 关键信息散落在多个文档中，汇总困难
- 人工撰写报告耗时长、格式不统一

#### Agent解决方案

**广西移动"AI智能办公助手"提供了标杆实践**[^126^]：

| 功能 | 能力描述 |
|------|----------|
| **"问"——办公知识库** | 覆盖规章制度、法务、市场、采购等16类知识库，沉淀6万+知识点，月均使用量超100万次 |
| **"写"——文案助手** | 覆盖报告、会议纪要、请示、通知等公文类型，预计覆盖80%办公文案场景 |
| **"办"——事务管家** | 会议预订、差旅安排、智能审批一键搞定 |
| **"查"——数据查询** | 自动抓取分析数据，生成报告大纲及可视化内容 |

**九天善智大模型的文档处理能力**[^54^][^191^]：
- 长文本智能化解析：支持超十万字文本理解
- 从多份文档中自动分析提炼关键信息
- 准确解析信息背后的核心观点
- 最终输出图文并茂、逻辑严谨的专业报告

#### 预期效果
- 文档阅读效率提升70%以上
- 报告撰写时间从数小时缩短至分钟级
- 知识查询从"耗时搜"变成"即时答"

### 5.2 场景二：PPT/Excel/Word自动化

#### 场景描述
各级管理人员需要频繁制作汇报PPT、数据分析Excel、公文Word等材料，包括月度经营分析、年度工作总结、项目汇报等。

#### 当前痛点
- 制作一份汇报PPT需要4-8小时
- 数据从多个系统导出后需要手动整理
- 图表制作耗时且格式不统一
- 材料内容更新后需要重新排版

#### Agent解决方案

**中国移动已推出支持15类公文写作的AI云电脑和自动生成PPT的智能鼠标**[^128^][^129^]：

- AI云电脑：内置15类公文写作模板，支持智能排版
- 智能鼠标：根据主题自动生成PPT大纲和内容
- 结构化数据处理：支持大规模多类型结构化数据分钟级处理分析[^191^]

**九天大模型的结构化数据洞察能力**[^191^]：
- 支持大规模多类型结构化数据分钟级处理分析
- 具备复杂数据理解和逻辑推理能力
- 全面支持分类、回归、预测、异常检测等任务
- 零样本或少样本情况下即可实现跨域应用

#### 预期效果
- PPT制作时间从4-8小时缩短至30分钟内
- 数据整理自动化率80%以上
- 图表生成实现一键化

### 5.3 场景三：会议记录与待办事项跟踪

#### 场景描述
中国移动各级会议频繁，从日常部门例会到重大决策会议，会后的纪要整理和待办跟踪耗费大量人力。

#### 当前痛点
- 会后整理纪要需要2-3小时
- 容易遗漏重要讨论要点
- 待办事项跟踪依赖人工提醒
- 历史会议记录检索困难

#### Agent解决方案

**中国移动云视讯AI慧记**[^133^]：
- 语音识别准确率高达95%
- 将语音内容转化为文本并自动生成会议纪要
- 采用多模态AI技术，结合语音、文本及图像信息
- 支持多种语言实时翻译

**移动云盘AI会议纪要功能**[^125^]：
- 实时将发言转成文字
- 自动标记重点内容
- 识别不同发言人的声音
- 自动生成结构化会议记录和待办事项清单

## 八、内网环境的特殊挑战与解决方案

### 8.1 中国移动内网环境特点

作为特大型央企，中国移动的IT架构具有以下显著特点：

1. **内外网物理隔离或强逻辑隔离**：生产系统、办公系统、管理系统分布在不同的网络区域[^43^]
2. **信创国产化要求**：全面适配国产芯片（龙芯、飞腾）、国产操作系统（统信UOS、麒麟）、国产数据库[^113^][^184^]
3. **安全合规等级高**：等保三级、涉密信息系统产品检测、数据不出域[^113^]
4. **数千个异构IT系统**：历史遗留系统多、接口不统一[^29^]

### 8.2 挑战一：模型底座的限制

#### 挑战描述
内网隔离环境下，无法直接访问OpenAI、Claude等海外大模型API，甚至部分国内公网大模型服务也无法直接调用。必须使用自有或本地部署的大模型。

#### 解决方案

**中国移动自研九天大模型体系**[^26^][^32^][^94^]：
- 全栈自主可控，从数据到推理全链路国产化
- 适配11个厂家17款国产AI芯片[^54^]
- 支持"云边端"灵活部署
- 九天安全可信大模型JT4.0版本[^90^]

**MoMA模型服务平台**[^27^][^88^]：
- 即使在内网环境下，也可通过MoMA平台调用300+模型
- 模型联邦与智能路由技术
- 单位Token成本压降30%以上
- "机密模型"服务基于硬件隔离保障数据安全

**私有化部署方案**[^42^][^44^]：
- 通过Ollama等工具在内网离线部署开源模型
- Docker容器化部署，一键启动
- 数据全程在内部网络完成，安全无泄漏

### 8.3 挑战二：Skills/Pip源的限制

#### 挑战描述
内网环境无法访问Python官方Pip源、npm等公网包管理器，AI Agent所需的依赖库、组件工具无法正常安装。

#### 解决方案

**聚智平台的组件化设计**[^55^]：
- 统一组件仓库：所有组件在平台内沉淀汇聚
- 标准化技能包：内置超150个行业Skills（MobileClaw框架）[^26^]
- 离线安装包：所有依赖提前打包在内部镜像源
- 本地化部署支持：平台支持全量内网部署

**内部镜像源建设**[^113^]：
- 建立内部Pip/Npm镜像站
- 定期同步公网依赖包
- 安全审计后导入内网
- 组件版本统一管控

### 8.4 挑战三：浏览器自动化的重要性

#### 挑战描述
中国移动内部大量系统（尤其是老系统）只能通过浏览器登录操作，没有API接口。Agent必须通过浏览器自动化来完成这些系统的数据提取和操作。

#### 解决方案

**磐匠RPA的浏览器自动化能力**[^51^][^66^][^172^]：
- 融合RPA+大模型+IDP+GSUT技术
- 能模仿日常操作自动化调用各类应用软件
- 自主识别系统界面、自主学习软件操作
- 无需改造现有系统即可部署

**RPA浏览器自动化的典型应用**[^134^][^136^][^141^]：

```
【浏览器自动化流程示例】
1. 打开浏览器 → 访问内部系统登录页
2. 自动输入用户名密码（或调用单点登录）
3. 导航到指定功能页面
4. 自动填写查询条件
5. 点击查询按钮
6. 提取页面数据
7. 导出到Excel/数据库
8. 关闭浏览器或跳转到下一个系统
```

**核心优势：**
- 无需改造现有系统：通过UI层操作，不需要系统提供API[^187^]
- 7×24小时不间断运行
- 操作可追溯、可审计
- 大幅降低人工错误率
- 支持跨系统数据流转

### 8.5 安全合规要求

#### 核心要求
- **数据不出域**：所有数据处理在内部网络完成[^43^]
- **操作可审计**：每一步操作都有日志记录

| 人机交互 | 无，纯自动执行 | 自然语言交互，人机协同 |
| 学习进化 | 无 | 持续学习优化 |

**融合演进路线**[^172^][^173^]：
1. **当前阶段（辅助驾驶）**：RPA执行，Agent决策，人工监督
2. **下一阶段（全自动驾驶）**：Agent自主拆解任务、自动寻找资源、自主选择工具甚至制作工具

### 9.3 "碳硅并举"的新型劳动模式

中国移动提出的"碳硅并举"理念[^41^][^42^]：
- **碳基员工**（人类）：负责创造性、决策性、情感性工作
- **硅基员工**（数字员工Agent）：负责重复性、规则性、计算性工作
- **协同方式**：人机协同，各取所长

## 十一、总结与展望

### 11.1 核心发现

1. **中国移动已构建完整的AI Agent技术体系**：从底座（聚智平台+九天大模型）到执行（磐匠RPA）到应用（智能体超市），形成全栈能力。

2. **规模化落地成效显著**：17000+数字员工上岗、350+生产场景覆盖、年减负70万人天，证明Agent在运营商内部具有巨大价值。

3. **"大模型+RPA"是内网环境的最优解**：在无法访问公网API的环境下，自研大模型+浏览器自动化RPA是实现AI Agent的可靠路径。

4. **Agent不是替代人类，而是增强**："碳硅并举"模式下，Agent承担重复性工作，人类专注创造性工作。

### 11.2 三大关键要点

| 要点 | 重要性 | 中国移动方案 |
|------|--------|-------------|
| **模型底座** | ★★★★★ | 九天大模型（自研自主可控）+ MoMA平台（300+模型联邦） |
| **Skills/组件源** | ★★★★ | 聚智平台组件仓库（150+行业Skills）+ 内部镜像源 |
| **浏览器自动化** | ★★★★★ | 磐匠RPA（融合大模型+GSUT，无需改造现有系统） |

### 11.3 未来演进方向

根据中国移动信息技术中心副总经理陈国的判断[^172^][^173^]：

1. **从辅助驾驶到全自动驾驶**：Agent将自主拆解任务、自动寻找资源、自主选择工具甚至制作工具
2. **从单智能体到多智能体协同**：复杂任务由多个Agent协同完成
3. **从工具到生产力主体**：数字员工成为"硅基同事"，深度融入生产流程
4. **从企业内向行业外输出**：将成熟能力赋能千行百业

### 11.4 对非技术人员的建议

1. **不必关心技术细节**：聚智平台和磐匠平台都提供低代码/无代码方式，业务人员可以像搭积木一样使用Agent
2. **从简单场景开始**：建议从文档处理、数据汇总等高频重复场景开始试点
3. **Agent是你的助手**：Agent不会取代你，而是帮你从繁琐工作中解放出来，专注于更有价值的创造性工作
4. **拥抱变化**：AI Agent技术正在快速演进，保持开放心态，积极学习新工具的使用方法
